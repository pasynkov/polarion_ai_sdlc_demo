## Functional Safety, Cybersecurity, and Fault Handling Software Requirements

The software shall implement functional safety requirements derived from the HARA and FMEA analyses, targeting the required SIL level (IEC 61508) or PL level (ISO 13849), and shall respect the defined Fault Tolerance Time Intervals (FTTi) for all safety-relevant functions. The software shall detect, classify, and react to faults according to defined error levels (Warning, Limited operation, Safe state), report errors using SPN/FMI codes, display blink codes on the HMI, and meet the targeted cybersecurity level through secure communication and access control on service interfaces.
