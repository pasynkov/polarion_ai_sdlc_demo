## Define Safety, Security, and Fault Handling Requirements from HARA and FMEA

Document all functional safety requirements derived from HARA and FMEA results, including: targeted SIL/PL levels, FTTi values per safety function, safe state definitions, fault detection and reaction matrix (fault → SPN/FMI code → error level → blink code → reaction), cybersecurity level target with access control requirements for service interfaces, and a traceability table linking each safety requirement back to its HARA/FMEA entry.
